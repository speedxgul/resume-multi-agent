# outputs/

This folder is generated by `python -m src.main update` (or `render`). All files here are derived — the source of truth is `resume.json`.

| File | Created by | Description |
|------|-----------|-------------|
| `resume.draft.json` | `update` | Raw synthesizer output **before** review. Useful for debugging or comparing to the final approved version. |
| `resume.json` | `update`, `render`, `shell save` | **Source of truth.** The approved structured resume. Edit this directly for fine-grained changes, then run `render` or `save`+`pdf` in the shell. |
| `resume.tex` | `update`, `render`, `shell save` | Generated LaTeX source. Regenerated from `resume.json` on every `save` / `render`. |
| `resume.pdf` | `update`, `compile`, `shell pdf` | Compiled PDF. Requires `tectonic` or `pdflatex`. |
| `resume.log` | `compile` | LaTeX compiler log. Check here if `pdf` fails. |

## Editing the resume

**Quick edits (typos, dates, exact wording):** Open `resume.json` in your editor, make changes, save, then in the shell:
```
> save
> pdf
```

**AI-assisted edits (rephrase, add bullets, reorder):** Use the edit shell:
```bash
python -m src.main shell
> edit experience
```

**Regenerate everything from JSON:**
```bash
python -m src.main render --open
```

## What is gitignored

The `.gitignore` excludes:
- `*.pdf`, `*.log`, `*.aux` and other LaTeX artifacts — binary/build outputs
- The JSON and TeX files are **not** ignored and can be committed if desired

Your `inputs/*.pdf`, `inputs/linkedin_profile.txt`, `inputs/twitter_profile.txt`, `inputs/manual_context.md`, and `inputs/urls.txt` are also gitignored (personal data).
